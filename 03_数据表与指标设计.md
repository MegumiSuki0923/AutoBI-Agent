# 数据表与指标设计

## 数据场景

项目使用模拟汽车营销销售数据，覆盖从投放到成交的业务链路：

广告投放 -> 销售线索 -> 试驾预约 -> 到店试驾 -> 成交购车。

数据不使用真实企业数据，重点模拟企业数据平台中的表结构、指标口径和查询分析流程。

## 数据表设计

### `dim_vehicle_model` 车型维表

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `model_id` | string | 车型 ID |
| `model_name` | string | 车型名称，如 XC60、XC90、EX30 |
| `series` | string | 车系 |
| `energy_type` | string | 燃油、插混、纯电 |
| `price_level` | string | 价格带 |
| `launch_year` | integer | 上市年份 |

示例数据：

| model_id | model_name | series | energy_type | price_level | launch_year |
| --- | --- | --- | --- | --- | --- |
| M001 | XC60 | SUV | 燃油 | 30-40 万 | 2018 |
| M002 | XC90 | SUV | 插混 | 60-80 万 | 2015 |
| M003 | EX30 | SUV | 纯电 | 20-30 万 | 2024 |

## 表关系

```text

  -> fact_sales.model_id
  -> fact_leads.model_id
  -> fact_test_drives.model_id
  -> fact_campaign_spend.model_id

fact_leads.lead_id
  -> fact_test_drives.lead_id
```

## 指标口径

### 销量

```sql
SUM(fact_sales.units_sold)
```

用于回答：

- 各地区销量排名。
- 不同车型销量趋势。
- 新能源车型销量占比。

### 销售额

```sql
SUM(fact_sales.sales_amount)
```

用于回答：

- 月度销售额趋势。
- 区域销售额对比。
- 车型销售额贡献。

### 有效线索数

```sql
COUNT(*) WHERE fact_leads.is_valid = true
```

用于回答：

- 各渠道有效线索数。
- 不同地区线索质量。

### 线索转试驾率

```sql
SUM(CASE WHEN converted_to_test_drive THEN 1 ELSE 0 END)
/ NULLIF(COUNT(*), 0)
```

用于回答：

- 哪个渠道线索更容易转化为试驾。
- 某地区线索转化率是否下降。

### 试驾完成率

```sql
COUNT(*) WHERE status = 'completed'
/ NULLIF(COUNT(*), 0)
```

用于回答：

- 哪些区域预约后取消率高。
- 哪些车型试驾完成率较低。

### 试驾到成交转化率

```sql
SUM(CASE WHEN converted_to_sale THEN 1 ELSE 0 END)
/ NULLIF(COUNT(*) WHERE status = 'completed', 0)
```

用于回答：

- 试驾后成交率。
- 高意向车型的成交表现。

### 点击率 CTR

```sql
SUM(clicks) / NULLIF(SUM(impressions), 0)
```

用于回答：

- 渠道曝光质量。
- 不同区域投放点击率。

### 单线索成本 CPL

```sql
SUM(spend) / NULLIF(SUM(leads_generated), 0)
```

用于回答：

- 渠道投放效率。
- 投放预算优化。
