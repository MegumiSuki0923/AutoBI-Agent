# AutoBI Agent：面向汽车营销数据平台的智能问数与分析助手

## 项目名称

**AutoBI Agent：面向汽车营销数据平台的智能问数与分析助手**

英文名称：

**AutoBI Agent: An LLM-powered Natural Language BI Assistant for Automotive Marketing Data Platform**

## 项目定位

AutoBI Agent 是一个面向汽车企业营销与销售数据场景的企业级问数智能体原型。项目目标是让业务人员通过自然语言查询结构化业务数据，系统自动完成问题理解、指标口径检索、Text-to-SQL、SQL 安全校验、数据查询、业务结论生成与图表展示。

项目解决的问题是：在企业数据平台中，业务人员往往不熟悉 SQL，也不清楚不同指标的计算口径，导致日常数据分析依赖数据分析师或研发人员。AutoBI Agent 通过 LLM + RAG + SQL Executor 的方式，把自然语言问题转化为可执行、可校验、可解释的数据查询流程。

## 与沃尔沃岗位的匹配点

该项目贴合 Digital AI Coding Intern / 数字化 AI Coding 实习生岗位的核心关键词：

- **AI Coding**：使用 AI 辅助完成接口设计、测试用例、Prompt 模板、技术文档和代码实现。
- **企业数字化**：围绕汽车营销和销售业务构建可落地的数据问数工具。
- **数据平台 / EDH**：用 DuckDB 或 SQLite 模拟企业数据集市，沉淀表结构、指标口径和数据字典。
- **数据接口**：基于 FastAPI + Pydantic 设计结构化问数接口。
- **问数智能体**：实现自然语言问题到 SQL 查询、分析总结和图表生成的闭环。
- **Prompt 规范**：沉淀意图识别、Text-to-SQL、SQL 校验、结果分析、图表推荐等 Prompt 模板。
- **AI 测试用例**：构建自然语言问数测试集，验证 SQL 正确性、安全性和分析结果质量。
- **技术文档沉淀**：输出接口文档、数据字典、指标口径文档、Prompt 规范和项目复盘。

## 业务场景

建议聚焦 **汽车营销销售漏斗分析场景**：

广告投放 -> 销售线索 -> 试驾预约 -> 到店试驾 -> 成交购车。

可支持的典型问题：

- 2026 年 4 月各区域 XC60 销量排名如何？
- 华东地区本月线索转化率相比上月下降了吗？
- 哪个渠道带来的试驾预约最多？
- 抖音投放的单线索成本 CPL 是否高于小红书？
- XC90 在一线城市的试驾到成交转化率是多少？
- 近 6 个月新能源车型销量趋势如何？

该场景天然覆盖数据表、指标口径、业务分析、可视化和异常解释，适合在简历和面试中展开。

## 核心功能模块

| 模块 | 功能 |
| --- | --- |
| 自然语言问题理解 | 识别用户想问的是销量、线索、转化率、投放成本、趋势还是排名 |
| 数据字典 / 指标口径管理 | 维护字段含义、表关系、指标公式，例如转化率、CPL、试驾率 |
| RAG 检索 | 根据问题检索相关表结构、字段说明、指标定义、示例 SQL |
| Text-to-SQL 查询 | 基于问题和检索上下文生成只读 SQL |
| SQL 安全校验 | 禁止 DROP/DELETE/UPDATE/INSERT，只允许 SELECT，限制表白名单和返回行数 |
| 数据查询执行 | 使用 DuckDB 或 SQLite 执行 SQL，返回结构化结果 |
| 结果分析总结 | LLM 根据查询结果生成业务解释、关键发现和行动建议 |
| 图表推荐与生成 | 根据数据形态推荐折线图、柱状图、饼图、漏斗图，并输出图表配置 |
| 历史记录管理 | 保存用户问题、SQL、结果摘要、图表类型、调用时间 |
| Prompt 模板沉淀 | 按意图识别、SQL 生成、SQL 校验、分析总结等分类管理 Prompt |
| AI 测试用例生成 | 自动生成典型问数问题、期望 SQL 片段、异常问题、边界测试 |

## 技术栈建议

| 层级 | 技术 |
| --- | --- |
| 后端 | Python 3.10+、FastAPI、Pydantic |
| 数据库 | DuckDB 优先，也可 SQLite |
| LLM | 原生 LLM API，或 LangChain 轻量封装 |
| RAG | Chroma / FAISS / SQLite FTS5，存储数据字典和指标文档 |
| SQL 解析校验 | sqlglot |
| 数据处理 | pandas |
| 前端 | Streamlit，或简单 Vue/React + ECharts |
| 可视化 | ECharts / Plotly |
| 测试 | pytest |
| 文档 | Markdown、OpenAPI、Prompt README |
| 工程管理 | Git、requirements.txt、.env.example |

推荐最小可行组合：

**FastAPI + DuckDB + Pydantic + 原生 LLM API + Chroma + Streamlit + pytest**
