# AutoBI Agent：面向汽车营销数据平台的智能问数与分析助手

AutoBI Agent 是一个面向汽车营销与销售数据场景的自然语言问数智能体。用户可以通过自然语言提出业务数据问题，系统自动理解问题意图，查询结构化数据，并生成分析结论和可视化结果。

## 项目定位

本项目模拟企业数据平台中的智能问数场景，面向汽车营销、销售和运营数据，构建一个轻量级 AI BI 助手。

项目核心目标是降低业务人员使用数据的门槛，使非技术用户无需编写 SQL，也能完成车型销量、线索转化、试驾预约、渠道投放效果等常见业务分析。

系统将围绕以下流程实现：

自然语言问题输入 -> 问题意图识别 -> 数据字典 / 指标口径检索 -> Text-to-SQL -> SQL 安全校验 -> 数据查询 -> 分析总结 -> 图表展示

该项目重点体现 AI Coding、企业数据平台、问数智能体、Prompt 规范、AI 测试用例和技术文档沉淀能力。

## 目标用户

| 目标用户             | 关注点 / 使用场景                                            |
| -------------------- | ------------------------------------------------------------ |
| 汽车企业市场运营人员 | 关注不同渠道投放效果、线索质量、获客成本和活动转化表现       |
| 销售分析人员         | 关注车型销量、区域销量排名、销售额趋势和成交转化情况         |
| 区域业务负责人       | 关注不同地区、城市等级、经销商的业务表现和异常波动           |
| 数据分析师           | 希望通过问数智能体提升日常取数、指标查询和分析报告生成效率   |
| AI 应用开发人员      | 关注 LLM、RAG、Text-to-SQL、SQL 安全校验在企业数据平台中的工程化落地方式 |

## 10 个标准业务问题

本项目 MVP 阶段优先支持以下标准业务问题：

1. 2026 年 4 月各区域 XC60 销量排名如何？
2. 近 6 个月 EX30 的销量趋势如何？
3. 2026 年 4 月各车型销量占比是多少？
4. 华东地区本月线索转试驾率是多少？
5. 哪个渠道带来的有效线索最多？
6. 抖音投放的单线索成本 CPL 是否高于小红书？
7. 各区域试驾预约完成率排名如何？
8. XC90 的试驾到成交转化率是多少？
9. 近 3 个月华南地区销售额变化趋势如何？
10. 哪个车型在一线城市的成交表现最好？

## MVP 范围

MVP 阶段目标是完成一个可运行、可演示、可测试的最小版本。

当前计划实现：

- 构建模拟汽车营销销售数据集。
- 设计车型、销售、线索、试驾、渠道投放等核心数据表。
- 支持用户输入自然语言业务问题。
- 基于数据字典和指标口径检索相关上下文。
- 使用 Text-to-SQL 将自然语言问题转换为 SQL。
- 对生成的 SQL 进行安全校验，只允许只读查询。
- 执行 SQL 并返回结构化查询结果。
- 根据查询结果生成简短业务分析结论。
- 根据结果数据推荐基础图表类型。
- 沉淀 Prompt 模板、接口说明、指标口径和测试用例文档。

MVP 阶段暂不实现：

- 真实企业数据接入。
- 企业账号、权限和审批系统。
- 复杂多轮对话。
- 生产级 BI 平台集成。
- 大规模分布式数据处理。
- 完整数据血缘和指标治理平台。

MVP 验收标准：

- 用户输入 10 个标准业务问题中的任意问题，系统能够返回 SQL、查询结果、分析结论和图表建议。
- 危险 SQL 或非查询类请求能够被安全拦截。
- 项目文档中包含 README、数据字典、指标口径、Prompt 模板和测试用例说明。
