# AutoBI Agent：面向汽车产业数据平台的智能问数助手

AutoBI Agent 是一个面向汽车产业多源数据的自然语言问数智能体。项目基于 `data/raw_data` 中的真实汽车产业 Excel 数据，完成数据盘点、字段标准化、指标口径沉淀和 DuckDB 入库，并构建 RAG + Text-to-SQL 问数链路，使业务用户能够用自然语言查询品牌产销、新能源渗透率、充电设施、动力电池装车量等产业指标。

## 项目定位

本项目面向企业数据平台中的智能问数场景。原始数据来自多个汽车产业 Excel 文件，主题覆盖汽车产销、新能源汽车产销、充电基础设施、动力电池和汽车上市公司经营数据。项目重点不是单纯做数据分析，而是把杂乱的多源数据整理成可查询、可解释、可测试的数据平台资产。

核心流程：

```text
原始 Excel 数据 -> 数据盘点 -> 字段标准化 -> 指标口径沉淀 -> DuckDB 入库
-> 自然语言问题输入 -> RAG 检索数据字典和指标口径 -> Text-to-SQL
-> SQL 安全校验 -> 数据查询 -> 结果分析 -> 图表展示
```

项目重点体现：

- AI Coding 辅助数据清洗、接口开发、Prompt 设计、测试用例和技术文档沉淀。
- 面向企业数据平台的数据接入、数据建模、指标口径管理和问数接口设计。
- 基于 RAG + Text-to-SQL 的自然语言问数智能体能力。
- SQL 安全校验、标准问题集验证和高质量软件交付意识。

## 目标用户

| 目标用户 | 关注点 / 使用场景 |
| --- | --- |
| 汽车产业分析人员 | 关注品牌产销、车型销量、新能源渗透率和产业趋势变化 |

## MVP 标准业务问题

1. 2022 年各厂商新能源汽车销量排名如何？
2. 比亚迪 2021-2022 年新能源汽车销量趋势如何？
3. 纯电动和插电式混合动力车型的销量结构有什么变化？
4. 2022 年新能源汽车渗透率的月度趋势如何？
5. 2022 年哪些车型销量最高？
6. 特斯拉上海 Model3 的月度销量趋势如何？
7. 汽车产量和销量是否同步增长？
8. 哪些省份的充电设施数量增长最快？
9. 不同材料类型动力电池装车量占比如何变化？
10. 不同车型类别的动力电池装车量趋势如何？

## MVP 范围

当前计划实现：

- 盘点 `data/raw_data` 下的汽车产业 Excel 数据。
- 选择汽车产销、新能源汽车产销、充电设施、动力电池 4 类数据作为 MVP 数据域。
- 将多源 Excel 字段标准化为统一英文表名和字段名。
- 将清洗后的结构化数据写入 `data/autobi.duckdb`。
- 沉淀数据字典和指标口径文档，作为 RAG 检索知识库。
- 支持用户输入自然语言产业问数问题。
- 基于数据字典和指标口径检索相关上下文。
- 使用 Text-to-SQL 将自然语言问题转换为只读 SQL。
- 对生成 SQL 进行安全校验，只允许白名单表的 `SELECT` 查询。
- 执行 SQL 并返回结构化查询结果。
- 根据查询结果生成简短业务分析结论。
- 根据结果数据推荐基础图表类型。
- 沉淀 Prompt 模板、接口说明、指标口径和测试用例文档。

暂不实现：

- 真实企业生产数据库接入。
- 企业账号、权限和审批系统。
- 复杂多轮对话。
- 生产级 BI 平台集成。
- 大规模分布式数据处理。
- 完整数据血缘和指标治理平台。
- 客户个人信息、线索、试驾、广告投放等当前原始数据不包含的营销漏斗场景。

## 验收标准

- 原始 Excel 数据能够被清洗并写入 DuckDB。
- DuckDB 中至少包含汽车产销、新能源汽车产销、充电设施、动力电池相关数据表。
- 用户输入标准业务问题时，系统能够返回 SQL、查询结果、分析结论和图表建议。
- 危险 SQL 或非查询类请求能够被安全拦截。
- 项目文档中包含 README、数据字典、指标口径、Prompt 模板和测试用例说明。
