# 两到三周开发计划

## 开发目标

在 2-3 周内完成一个可演示、可测试、可写进简历的 AutoBI Agent 原型，重点展示：

- 汽车业务数据建模能力。
- 企业数据平台接口设计能力。
- LLM + RAG + Text-to-SQL 应用开发能力。
- SQL 安全与测试验证意识。
- Prompt、接口、指标和技术文档沉淀能力。

## 第 1 阶段：业务与数据基础

### Day 1：明确项目边界

任务：

- 确定项目名称、目标用户和典型问数场景。
- 明确最小可行版本范围。
- 写 README 初稿。

产出：

- `README.md`
- 功能清单
- 10 个标准业务问题

### Day 2：设计数据表和指标口径

任务：

- 设计 5 张业务表。
- 编写字段说明。
- 定义销量、销售额、转化率、CTR、CPL 等指标。

产出：

- `docs/data_dictionary.md`
- `docs/metrics.md`

### Day 3：生成模拟数据

任务：

- 用 Python 生成 6-12 个月模拟数据。
- 写入 DuckDB 或 SQLite。
- 检查数据分布是否合理。

产出：

- `data/autodata.duckdb`
- `scripts/generate_mock_data.py`

## 第 2 阶段：后端问数链路

### Day 4：搭建 FastAPI 项目

任务：

- 初始化项目结构。
- 编写 Pydantic 请求和响应模型。
- 实现 `/api/ask` 占位接口。

产出：

- `app/main.py`
- `app/schemas.py`
- `/api/ask`

### Day 5：实现数据库查询层

任务：

- 封装 DuckDB 连接。
- 实现 SQL 执行器。
- 返回 columns + rows 结构。

产出：

- `sql_executor.py`
- 查询层单元测试

### Day 6：实现 RAG 文档检索

任务：

- 将数据字典、指标口径、示例 SQL 切分为文档。
- 构建向量索引或 FTS 检索。
- 根据用户问题返回相关上下文。

产出：

- `rag_service.py`
- schema / metric 检索 Demo

### Day 7：实现 Text-to-SQL 模块

任务：

- 编写 Text-to-SQL Prompt。
- 接入 LLM API。
- 规范 JSON 输出格式。

产出：

- `text_to_sql_service.py`
- `prompts/text_to_sql_prompt.md`

### Day 8：实现 SQL 安全校验

任务：

- 使用 sqlglot 解析 SQL。
- 禁止非 SELECT 操作。
- 限制表白名单。
- 自动补充 LIMIT。

产出：

- `sql_guard.py`
- `tests/test_sql_guard.py`

## 第 3 阶段：分析、前端与工程沉淀

### Day 9：实现分析总结模块

任务：

- 编写结果分析 Prompt。
- 输入用户问题、SQL 和查询结果。
- 输出核心结论、数据依据和行动建议。

产出：

- `analysis_service.py`
- `prompts/analysis_prompt.md`

### Day 10：实现图表推荐模块

任务：

- 根据字段类型和问题意图推荐图表。
- 输出 ECharts 或 Plotly 配置。

产出：

- `chart_service.py`
- 图表配置 JSON

### Day 11：搭建 Streamlit 前端

任务：

- 实现问题输入框。
- 展示 SQL、表格、图表和分析结论。
- 增加示例问题按钮。

产出：

- `frontend/streamlit_app.py`
- 可演示页面

### Day 12：实现历史记录管理

任务：

- 保存用户问题、SQL、结果摘要和图表配置。
- 支持查看最近 N 条查询。

产出：

- `history_service.py`
- query history 页面或接口

### Day 13：编写测试集

任务：

- 编写标准问数测试。
- 编写 SQL 安全测试。
- 编写边界问题测试。

产出：

- `tests/test_api_ask.py`
- `tests/test_text_to_sql_cases.py`
- `docs/test_cases.md`

### Day 14：整理技术文档

任务：

- 整理 Prompt 模板。
- 整理接口文档。
- 整理数据字典。
- 补充项目复盘。

产出：

- `docs/prompt_templates.md`
- `docs/api_design.md`
- `docs/project_summary.md`

### Day 15：简历与面试材料

任务：

- 录制或截图 Demo。
- 整理简历 bullet。
- 准备面试讲解稿。

产出：

- 项目截图
- 简历项目经历
- 面试 Q&A

## 2 周压缩版

如果时间只有 2 周，优先完成：

- DuckDB 模拟数据。
- FastAPI 主问数接口。
- RAG + Text-to-SQL。
- SQL Guard。
- Streamlit 展示。
- Prompt 文档。
- pytest 核心测试。

可以暂缓：

- 多轮对话。
- 复杂历史记录。
- 前端美化。
- 多 Agent 规划。
- 真实权限系统。

## 每日验收标准

| 阶段 | 验收标准 |
| --- | --- |
| 数据阶段 | 能用 SQL 查出销量、线索、试驾、投放等指标 |
| 后端阶段 | `/api/ask` 能返回 SQL、结果和分析摘要 |
| 安全阶段 | 危险 SQL 能被拦截 |
| 前端阶段 | 用户能输入问题并看到图表 |
| 测试阶段 | pytest 能覆盖核心链路 |
| 文档阶段 | README、Prompt、指标、接口文档完整 |

## 推荐 Demo 问题

- 2026 年 4 月各区域 XC60 销量排名如何？
- 近 6 个月 EX30 销量趋势如何？
- 哪个渠道带来的有效线索最多？
- 华东地区本月线索转试驾率是多少？
- 抖音投放的 CPL 是否高于小红书？
- XC90 试驾到成交转化率是多少？
